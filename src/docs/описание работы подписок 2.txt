**Подписка на изменения параметров:**
   - В системе реализован механизм подписки на изменения параметров.
   - Каждый пользователь может выбирать, к каким параметрам или сущностям он хочет подписаться.

**Обновление по изменениям за определенный интервал:**
   - При подписке на параметр, пользователь может указать интервал времени, за который он хочет получать обновления.
   - Если параметр меняется в течение этого интервала, подписчик получит массив с изменениями за указанный период.

**Работа с историей:**
   - Реализована возможность получения истории изменений параметра за определенный период.
   - История может быть короткой (задается интервалом) или долговременной (задается датой начала и конца).

**Добавление подписок:**
   - Для добавления новой подписки, пользователь может открыть настройки и выбрать соответствующий параметр.
   - Пользователь указывает имя подписки и выбирает сущности, за которыми он хочет наблюдать.

**Управление подписками:**
   - Пользователь может управлять своими подписками, изменяя их параметры или удаляя.
   - Каждая подписка имеет уникальное имя и связана с определенными сущностями.

**Изменение параметров и распределение обновлений:**
   - Изменения параметров моментально распространяются на всех подписчиков.
   - Подписчики получают обновления в режиме реального времени при изменении параметров, к которым они подписаны.

**Именование подписок:**
   - Для удобства пользователей предлагается использовать понятные имена для своих подписок.
   - Это помогает легко определить, к каким параметрам относится каждая подписка.

**Инструкция по созданию подписок:**

**1. Определение параметров подписок:**
   - Даже при одинаковых параметрах можно создавать разные подписки.
   - Возможность создания двух и более подписок.

**2. Именование подписок:**
   - Имя подписки является важным элементом.

**3. Создание списка сущностей в каждой подписке:**
   - В каждой подписке формируется список сущностей, на которые она подписана.
   - Сущности могут иметь различные параметры.

**4. Выбор параметров для подписок:**
   - Параметры подписки определяются в соответствии с интересами пользователя.

**5. Пример создания подписки:**
   - Подписка с именем "Подписка 1".
   - Подписка на сущность "entity 1".
   - Подписка только на переменную "A".

**6. Создание нескольких подписок на одну сущность:**
   - Возможность создания второй подписки на ту же сущность, но с другими параметрами (например, переменная "B").

**7. Настройка интерфейса:**
   - Размещение списка подписок и сущностей.
   - Возможность раскрытия каждой подписки для редактирования параметров.

**8. Добавление новых подписок:**
   - Кнопка "Добавить подписку" для удобства создания новых подписок.

**9. Управление подписками:**
   - Возможность открывать подписки из списка.
   - Возможность изменять параметры подписок.

**10. Использование подписок в скриптах:**
    - Связь между изменением параметра в подписке и вызовом соответствующего скрипта.

**11. Множественные скрипты для одной подписки:**
    - Возможность привязки нескольких скриптов к одной подписке для оптимизации и удобства использования.

**12. Управление подписками через настройки:**
    - Доступ к списку текущих подписок.
    - Возможность добавления новых подписок из настроек.

**13. Отображение подписок в интерфейсе:**
    - Удобное отображение списка подписок и выбор текущей подписки.

**14. Изменение параметров подписок:**
    - Возможность изменения интервала изменения истории, доступных сущностей и других параметров.

**15. Организация в виде вкладок:**
    - Предложение организации подписок в виде вкладок для лучшего управления.

**16. Добавление новых параметров:**
    - Возможность динамического добавления параметров в каждую подписку.

**17. Контроль над количеством вкладок:**
    - Предложение управления числом вкладок в зависимости от числа подписок.

**18. Обзор изменений:**
    - Обеспечение возможности отслеживания изменений в параметрах подписок.

**19. Визуализация данных:**
    - Возможность визуализации данных в каждой подписке для более наглядного представления.
